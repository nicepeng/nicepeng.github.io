03 | Python基础语法详解

上一篇我们搭好了 Python 开发环境，成功写出了人生第一行代码 print("Hello World")。很多同学跑完程序后会好奇：

  • 为什么有的代码前面要空几格？不空就直接报错？
  • 变量名为什么不能随便起？有的名字一写就红名报错？
  • 我写的这些代码，在电脑里到底是怎么运行的？

这些问题，本质上都属于 Python 的「基础语法规则」。就像写作文要遵守标点、分段规范，写代码也有一套固定的规则 —— 规则不多，但非常严格，写错一个细节，程序就会直接罢工。

今天这篇文章，我们就把 Python 入门最核心的基础语法一次性讲透。全程大白话讲解 + 可运行代码示例 + 新手避坑指南，看完你不仅能看懂代码的基本结构，还能知道「为什么这么写」，写出规范、不易出错的代码。

一、Python 最独特的规则：缩进（代码块的灵魂）

如果说 Python 和 C、Java、JavaScript 等语言最大的区别是什么，那一定是「缩进」。

① 其他语言怎么划分代码块？

在大多数编程语言里，代码块是用一对大括号 {} 包裹起来的。比如 Java 里的判断逻辑：

大括号里面的两行，就是满足条件才会执行的代码块。大括号是边界，缩进只是为了好看，不缩进也能运行。

② Python 用缩进划分代码块

Python 里没有大括号，完全靠缩进（行首的空格）来区分代码的层级和归属。同一级别的代码，缩进量必须完全一致。

  • 第 2 行 if 是顶层代码，前面没有空格；

  • 第 5 行又回到顶层，不管条件是否满足都会执行。

③ 缩进的硬性规则

推荐用 4 个空格作为一个缩进层级，这是 PEP8 官方规范的标准写法。

同一层级的代码，缩进量必须完全相同，多一个少一个都会报错。

Tab 和空格绝对不能混用，不然会报 IndentationError 缩进错误，新手最容易踩这个坑。

缩进不能随便加，顶层代码前面不能有空格。

💡 新手小技巧：VS Code 和 PyCharm 都默认开启了 Tab 自动转 4 空格，直接按 Tab 键就行，不用手动敲 4 下空格。

④ 常见缩进错误

  • 缩进不一致：同一代码块有的行 3 个空格，有的行 4 个
  • 该缩进的没缩进：if、for、def 等语句后面的代码块忘记缩进
  • 不该缩进的乱缩进：顶层代码前面莫名加了空格

缩进错误虽然常见，但也好排查 —— 报错信息会直接告诉你第几行缩进有问题，对着改就行。

二、标识符：给变量、函数起名字的规则

我们写代码时，会给变量、函数、类起名字，这些名字就叫做「标识符」。名字不能乱起，有一套明确的规则。

2.1 硬性规则（违反就报错）

  1. 只能由字母、数字、下划线 _ 组成，不能有空格、特殊符号（比如 @、#、￥ 都不行）。
  2. 不能以数字开头。比如 123name 是错误的，name123 是正确的。
  3. 严格区分大小写。Age 和 age 是两个完全不同的变量。
  4. 不能使用 Python 的关键字（保留字），下一节会讲。

2.2 软性规范（专业写法，PEP8 规范）

规则是必须遵守的，规范是让你的代码更专业、更好读的约定：

  1. 见名知意：尽量用有意义的英文单词，不要用 a、b、x 这种无意义的名字。比如存姓名用 name，不要用 n。
  2. 变量名、函数名用「蛇形命名法」：全部小写，多个单词之间用下划线连接。比如 user_name、student_age。
  3. 类名用「大驼峰命名法」：每个单词首字母大写，比如 StudentInfo，面向对象篇我们再细讲。
  4. 常量约定全大写：Python 没有真正的常量，我们约定全大写的变量就是常量，告诉别人这个值不要修改，比如 MAX_SIZE = 1024。

❌ 错误标识符举例

  • 123abc：数字开头
  • user name：有空格
  • user@name：有特殊符号
  • if：是关键字

三、关键字：Python 自带的「保留词」

关键字也叫保留字，是 Python 语言内部已经占用、有特殊功能和含义的单词。绝对不能用来当变量名、函数名、类名，不然会直接报错。

查看所有关键字：help("keywords")

运行后会列出所有关键字，还会按首字母分类。

常见关键字一览

  • 流程控制：if、elif、else、for、while、break、continue
  • 函数与类：def、return、class、pass
  • 逻辑与值：True、False、None、and、or、not、in、is
  • 异常处理：try、except、finally、raise
  • 其他：import、from、as、global、nonlocal、lambda、yield

⚠️ 新手避坑

  1. True、False、None 首字母都是大写，写成小写 true、false 会报错，这是新手最常犯的错误之一。
  2. 不用刻意背所有关键字，你的代码编辑器会自动给关键字高亮变色，一看就知道是关键字，不会拿来当变量名。

四、注释：给人看的说明文字

代码是写给机器运行的，但更是写给人看的 —— 可能是同事，也可能是三个月后的你自己。注释就是用来说明代码的文字，程序运行时会直接忽略注释，不会执行。

4.1 三种注释方式

1）单行注释：# 开头

用 # 号开头，后面跟注释内容，从 # 开始到行尾都是注释。

可以单独占一行，用来解释下面的代码；也可以跟在代码后面，用来补充说明当前行。

2）多行注释：三引号包裹

如果注释内容很多，一行写不下，可以用三个单引号 ''' 或者三个双引号 """ 包裹起来，实现多行注释。

3）文档字符串（Docstring）

文档字符串本质上就是三引号注释，专门写在函数、类、模块的开头，用来说明这个函数 / 类的功能、参数、返回值。它有特殊作用，可以通过 help() 函数查看，后面函数篇我们会详细讲。

4.2 注释的最佳实践

  1. 注释是解释「为什么这么写」，而不是「写了什么」。比如 age = age + 1 不用写注释「年龄加 1」，代码本身已经说清楚了，你要写的是「为什么要加 1」。
  2. 不用每行都加注释，只在逻辑复杂、容易踩坑、业务特殊的地方加。
  3. 注释要和代码同步更新，代码改了注释也要改，不然过时的注释反而会误导人。
  4. 尽量用清晰的代码命名来减少注释，好的代码本身就是说明书。

五、变量：数据的「标签」

变量是编程里最基础的概念，简单来说，变量就是给数据起个名字，方便我们后面反复使用。

5.1 变量的本质：标签，不是盒子

很多教程会说「变量是装数据的盒子」，这个说法在 Python 里是不准确的。

  • 其他语言（比如 C 语言）的变量是盒子：你创建一个变量，就申请一块内存空间，把数据放进去。
  • Python 的变量是标签：数据本身存在内存里，变量只是贴在数据上的一个名字标签。一个数据可以贴多个标签，标签可以撕下来贴到别的数据上。

七、Python 内存管理与垃圾回收（入门理解版）

很多新手会好奇：我创建的变量、数据，都存在哪里？用完了要不要手动删除？会不会占内存？

这就涉及到 Python 的内存管理机制。简单来说：Python 会自动帮你管理内存，不用手动申请和释放。它有一套完整的垃圾回收（Garbage Collection，简称 GC）机制，自动清理没用的数据，释放内存。

7.1 数据在内存里的样子

我们写 age = 20 时，Python 做了两件事：

  1. 在内存里开辟一块空间，存储整数 20 这个数据；
  2. 创建变量名 age，把它贴在 20 这块内存空间上，也就是建立引用关系。

当我们写 age = 30 时：

  1. 在内存里又开辟一块新空间，存储整数 30；
  2. 把 age 标签从 20 上撕下来，贴到 30 上；
  3. 这时候 20 就没有任何变量引用它了，成了「垃圾」。

7.2 垃圾回收核心：引用计数

Python 垃圾回收以引用计数为主。简单来说，每个数据对象身上都有一个计数器，记录着有多少个变量引用（指向）它。

引用计数 +1 的情况

  • 变量被赋值：a = 10，10 的计数 +1
  • 变量被其他变量引用：b = a，10 的计数再 +1
  • 数据被放入容器（列表、字典、元组）：lst = [a, 20]，10 的计数 +1
  • 作为参数传入函数

引用计数 -1 的情况

  • 变量被删除：del a，10 的计数 -1
  • 变量被重新赋值：a = 20，原来的 10 计数 -1
  • 容器被删除，或者元素从容器中移除
  • 函数执行结束，局部变量被销毁

当引用计数变成 0 的时候，这个数据对象就会被立刻销毁，占用的内存空间被回收。

7.3 引用计数的缺陷：循环引用

引用计数简单高效，但有一个致命问题：循环引用（也叫交叉引用）。

举个例子：

删除 a 和 b 之后，两个列表的引用计数都变成 1（互相引用），但外面已经没有任何变量能访问到它们了。它们的引用计数永远不会变成 0，引用计数机制回收不了它们，就会造成内存泄漏。

7.4 补充机制：分代回收

为了解决循环引用的问题，Python 引入了分代回收作为补充。

分代回收的核心思想很简单：存活时间越长的对象，越不可能是垃圾，就越少去检查它。

Python 把所有对象分成三代：

  • 0 代（年轻代）：新创建的对象都是 0 代。检查最频繁，因为大部分新对象很快就会变成垃圾。
  • 1 代（中年代）：0 代对象经过一次垃圾回收还存活，就升级到 1 代。检查频率次之。
  • 2 代（老年代）：1 代对象经过一次垃圾回收还存活，就升级到 2 代。检查频率最低。

当某一代对象数量达到阈值时，就会触发这一代的垃圾回收，清理掉没用的对象，包括循环引用的对象。

7.5 新手要知道的几点

  1. 绝大多数情况下，你完全不用管内存管理，Python 自动帮你处理得很好。
  2. 尽量避免写循环引用的代码，减少内存泄漏的可能。
  3. 特别大的数据用完之后，可以手动 del 变量名 删除，加速回收（一般场景不需要）。
  4. 不用一开始就死磕底层原理，先会用代码，进阶阶段再深入研究。
