04 | 详解Python中的所有数据类型与运算符

上一篇文章我们搞定了 Python 的基础语法规则，知道了怎么定义变量、怎么和程序交互。但变量只是「贴在数据上的标签」，我们编程的核心，本质上就是对各类数据进行处理。

不同类型的数据，能做的操作、适用的场景完全不同；而运算符，就是操控这些数据的「基础工具」—— 加减乘除做计算，比较大小做判断，逻辑运算做条件组合。可以说，数据类型 + 运算符，是搭建所有 Python 代码的底层积木。

今天这篇文章，我们一次性把 Python 所有常用数据类型和运算符全部讲透，搭配可运行代码示例 + 新手避坑指南，看完就能上手用。

一、Python 数据类型总览

Python 是动态类型语言：变量不用提前声明类型，但数据本身有明确的类型。日常开发中 99% 会用到的类型，可以分成 8 大类：

大类 具体类型 核心特点 典型用途
数值类型 int / float / complex 存储数字，支持数学运算 计算、统计、数值处理
布尔类型 bool 仅 True/False 两个值 条件判断、逻辑分支
字符串类型 str 存储文本信息，不可变 文本处理、用户信息存储
序列类型 list（列表）/tuple（元组） 有序的元素集合，支持索引切片 存储一组有序数据
映射类型 dict（字典） 键值对形式存储，查找极快 存储对应关系、结构化数据
集合类型 set（集合） 无序、元素唯一，自动去重 数据去重、集合运算
特殊空类型 NoneType 仅 None 一个值，表示「空」 标记空值、默认值

下面我们逐个拆解，重点讲高频使用的核心类型，冷门特性点到为止。

1.1 数值类型：数字计算的基础

数值类型用来存储数字，是所有计算的基础，分为整数、浮点数、复数三类。

（1）整数 int

Python 里的整数没有大小限制，支持任意精度，哪怕是几百位的数字也能正常运算，这是 Python 相比其他语言的一大优势。

除了十进制，还支持其他进制表示（了解即可）：

  • 二进制：0b 开头，比如 0b101 对应十进制 5
  • 八进制：0o 开头，比如 0o10 对应十进制 8
  • 十六进制：0x 开头，比如 0x1A 对应十进制 26

**（2）**浮点数 float

浮点数就是我们说的小数，支持普通写法和科学计数法。

⚠️ 新手第一大坑：浮点数精度丢失

（3）复数 complex

由实部和虚部组成，格式为 a + bj，比如 3 + 4j。仅在科学计算、信号处理等专业场景使用，新手阶段几乎不会用到，了解即可。

数值类型转换

我们可以用内置函数实现类型转换：

  • int()：转整数，浮点数会直接截断小数部分，字符串只能转纯数字内容
  • float()：转浮点数

1.2 布尔类型 bool：判断的核心

布尔类型只有两个值：True（真）和 False（假），首字母必须大写，这是新手高频出错点。

核心本质：布尔值就是特殊的整数

在 Python 底层，True 本质就是整数 1，False 本质就是整数 0，所以它们可以直接参与数值运算：

真假值判断：所有数据都能当布尔值用

在条件判断、逻辑运算场景下，任何数据类型都能自动转换成布尔值。

  • 为假的值（固定 8 种）：0、0.0、None、空字符串 ""、空列表 []、空元组 ()、空字典 {}、空集合 set()
  • 为真的值：除了上面的，几乎所有值都为真

1.3 字符串类型 str：文本处理的核心

字符串用来存储文本内容，是 Python 里使用最频繁的类型之一。

（1）定义方式

有三种定义方式，各有适用场景：

① 单引号 / 双引号：单行字符串，两者完全等价

② 三引号：多行字符串，支持换行，也常用来写注释

💡 小技巧：字符串里要包含引号时，可以交叉使用，比如 "他说'你好'"，避免转义。

（2）转义字符

如果字符串里要输入特殊字符（比如换行、制表符、引号），需要用 \ 转义：

\t 制表符（Tab 键效果）
\ 表示反斜杠本身
' 表示单引号
" 表示双引号

1.4 列表 list：最常用的可变容器

列表是有序、可变的元素集合，是 Python 里使用最频繁的容器类型。用中括号 [] 定义，元素之间用逗号隔开。

核心特点

  1. 有序：每个元素有对应的索引（下标），从 0 开始，支持索引访问、切片
  2. 可变：可以随时增、删、改里面的元素
  3. 元素可重复：同一个值可以出现多次
  4. 支持嵌套：列表里可以再放列表，形成二维列表

基础操作

更多列表方法我们会在后续数据结构专题里详细讲。

1.5 元组 tuple：不可变的列表

元组和列表几乎一样，唯一的区别是：元组不可变，创建之后不能增、删、改元素。用小括号 () 定义。

关键注意点

元组的核心标识不是括号，而是逗号。只有一个元素的元组，必须加逗号，否则会被当成普通变量：

适用场景

因为不可变，元组比列表更安全、性能更好，适合：

  • 存储不需要修改的固定数据
  • 作为字典的键（列表不行，因为可变）
  • 函数返回多个值时，默认返回的就是元组

1.6 字典 dict：键值对映射容器

字典是无序的「键 - 值对」集合，最大的优势是查找速度极快（时间复杂度 O (1)），适合存储对应关系。用大括号 {} 定义，键和值用冒号 : 分隔。

核心规则

  1. 键的要求：必须是不可变类型（int、str、tuple 等），不能是列表；且键必须唯一，重复赋值会覆盖旧值
  2. 值的要求：没有任何限制，可以是任意类型
  3. 有序性：Python 3.7+ 版本字典会保留插入顺序，但本质还是映射类型，不推荐用索引访问

基础操作

1.7 集合 set：自动去重的无序容器

集合和数学里的集合概念一致，核心特性是无序、元素唯一，天生支持去重和集合交并差运算。用大括号 {} 定义。

关键注意点

  • 空集合必须用 set() 创建，不能用 {}——{} 默认是空字典
  • 集合元素必须是不可变类型，和字典的键规则一致
  • 集合无序，不支持索引访问

典型用途

  • 数据去重：把列表转成集合，自动去除重复元素
  • 集合运算：求两个数据集的交集、并集、差集

1.8 特殊空类型 NoneType

这个类型只有一个值：None，专门用来表示「空、没有值」的状态。

容易混淆的点

  • None 不是 0
  • None 不是空字符串 ""
  • None 不是 False

它是一个独立的、专门的空值类型，比如函数没有返回值时，默认返回的就是 None。

2.4 逻辑运算符：组合判断条件

用来组合多个布尔条件，一共有三个：and（与）、or（或）、not（非）。

运算符 含义 规则说明
and 与 两边都为真，结果才为真
or 或 只要有一边为真，结果就为真
not 非 取反，真变假，假变真

核心特性：短路逻辑

逻辑运算符有「短路」特性：一旦能确定最终结果，后面的代码就不会执行了，这是新手必须掌握的重点。

  • and 短路：左边为假时，直接返回左边的值，右边完全不执行
  • or 短路：左边为真时，直接返回左边的值，右边完全不执行

这个特性在写简化代码时非常实用。

2.5 成员运算符：判断是否包含

用来判断一个元素是否在某个容器（字符串、列表、元组、字典、集合）里，返回布尔值。

运算符 含义 示例
in 存在于… 中 “a” in “abc”
not in 不存在于… 中 “d” not in [1,2,3]

2.6 身份运算符：判断是否为同一个对象

用来判断两个变量是不是指向同一个内存对象（也就是内存地址是否相同），一共两个：is、is not。

高频混淆：is 和 == 的区别

  • ==：判断两个变量的值是否相等
  • is：判断两个变量的 身份（内存地址） 是否相同，也就是是不是同一个对象

特殊现象：小整数池

Python 会把 -5 ~ 256 之间的整数缓存起来，重复使用，所以这个范围内的相同整数，is 判断也为真：

但这是 Python 的内部优化机制，绝对不要依赖这个特性。日常开发中：

  • 判断数值、字符串等内容是否相等，一律用 ==
  • 只有判断一个变量是不是 None 时，才固定用 is

2.7 位运算符（了解即可）

位运算符是直接操作二进制位的，包括 &（按位与）、|（按位或）、^（按位异或）、~（按位取反）、<<（左移）、>>（右移）。

这类运算符在底层开发、算法优化中偶尔会用到，新手阶段几乎不会接触，了解有这个概念即可，不用深入。

2.8 运算符优先级总表

运算符优先级从高到低排序如下，不用死记硬背：

优先级 运算符分类 具体运算符
最高 括号 ()
↓ 幂运算 **
↓ 正负号 +x、-x
↓ 算术运算符 *、/、//、%
↓ 算术运算符 +、-
↓ 位运算符 <<、>>、&、`
↓ 比较运算符 ==、!=、>、<等
↓ 逻辑非 not
↓ 逻辑与 and
最低 逻辑或 or

✅ 最佳实践：如果不确定优先级，直接加括号就行。代码的可读性永远比炫技重要。

三、新手高频避坑总结

  1. 浮点数精度问题：不要直接用 == 判断两个浮点数是否相等，用差值小于某个极小值判断
  2. is 和 == 混用：判断值用 ==，判断 None 用 is
  3. 地板除负数陷阱：// 是向下取整，不是截断小数
  4. 空集合定义错误：空集合是 set()，{} 是空字典
  5. 逻辑运算返回值：and/or 返回的是操作数本身，不是布尔值
  6. 字符串不可变：不能直接修改字符串单个字符，需要重新生成

练习题 1：浮点数精度处理（基础进阶）

日常开发中经常需要处理金额，浮点数的精度误差会导致金额计算错误。请编写程序： 用户输入一个以「元」为单位的小数（比如 0.1、123.45），将其准确转换为以「分」为单位的整数，要求完全避免浮点数精度误差（例如输入 0.1 必须输出 10，不能输出 9）。

解题思路

浮点数精度误差的根源是二进制无法精确表示所有十进制小数，直接 int(金额 * 100) 会出现误差。

正确的解决方案有两种：

  • 字符串处理法：将输入的字符串按小数点分割，分别处理整数部分和小数部分，拼接后转整数，从根源避开浮点数运算
  • decimal 标准库法：使用 Python 内置的高精度十进制运算模块 decimal，专门用于金融等对精度要求高的场景

方法 1：字符串处理法（推荐新手理解原理）

方法 2：decimal 高精度法（生产环境推荐）

易错点分析

  1. 直接写 int(float(input()) * 100) 是新手最常见的错误，输入 0.1 时会得到 9，因为 0.1 * 100 在底层实际是 9.999999999999998，int 截断后变成 9
  2. 不能用 round() 处理金额，极端场景下依然会有精度误差，金融场景必须用 decimal 模块

练习题 3：身份与相等性深度辨析（中等难度）

针对以下 5 组变量，分别判断 a == b 和 a is b 的结果（True/False），并逐一说明底层原因，考察小整数缓存、字符串驻留、可变 / 不可变对象特性。

答案与解析

组别 a == b a is b 底层原因
组 1 TRUE TRUE Python 内置了小整数缓存池，范围是 -5 ~ 256，这个范围内的整数会被提前创建好，重复使用，所以相同数值指向同一个内存对象。
组 2 TRUE False（交互式环境下） 257 超出小整数缓存范围，交互式环境下每次赋值都会创建新的整数对象，所以值相等但身份不同。⚠️ 注意：在脚本文件（.py）中，Python 会做编译优化，同一行 / 同一代码块内的相同大整数也会复用，可能出现 a is b 为 True 的情况，但这是编译器优化，不是语言特性，开发中绝对不能依赖这个特性。
组 3 TRUE TRUE Python 有字符串驻留（intern）机制，对于符合标识符规则的字符串（仅包含字母、数字、下划线），会自动驻留，相同字符串只保存一份，所以指向同一个对象。
组 4 TRUE FALSE 列表是可变对象，每次用 [] 创建列表都会开辟新的内存空间，哪怕内容完全相同，也是两个独立的对象，所以值相等但身份不同。
组 5 TRUE 视环境而定（大多为 True） 元组是不可变对象，但 Python 不会对所有元组做缓存复用。小的、内容简单的元组在编译优化下可能会复用对象，is 结果为 True；但复杂元组可能不会。和大整数一样，不可变对象的 is 为 True 只是优化结果，不能作为开发依据，判断内容相等一律用 ==。

核心结论

  1. == 判断值是否相等，is 判断内存地址是否相同（是否为同一个对象）
  2. 只有判断 None 的时候，才固定用 is，其余场景判断内容一律用 ==
  3. 小整数池、字符串驻留都是 Python 的性能优化手段，绝对不能依赖这些特性写逻辑代码

练习题 5：变量交换的底层原理（原理辨析）

Python 中可以用 a, b = b, a 一行代码实现两个变量的值交换，这是 Python 的特色语法。请回答以下 3 个问题，并用代码验证你的结论：

  1. 这条语句的底层执行顺序是什么？右边的表达式会在什么时候计算？
  2. 如果 a 和 b 是列表（可变对象），交换后两个变量指向的内存地址会发生什么变化？
  3. 如果写成 a = b; b = a 为什么不能实现交换？和 a, b = b, a 的本质区别是什么？

问题 1：执行顺序

a, b = b, a 的执行顺序是：

  1. 先计算右边所有表达式的值：先取出 b 的值，再取出 a 的值，在内存中打包成一个临时元组 (b的值, a的值)
  2. 再进行解包赋值：将临时元组的第一个元素赋值给 a，第二个元素赋值给 b

核心：右边的所有值都会在赋值开始前全部计算完成，不会因为左边变量的修改而影响右边的取值。

问题 2：可变对象交换的内存变化

列表是可变对象，但交换的是变量的指向（标签），不是列表本身的内容。交换后，a 会指向原来 b 指向的列表对象，b 会指向原来 a 指向的列表对象，两个列表本身的内存地址不会变化，只是变量的指向发生了交换。

问题 3：a = b; b = a 为什么不能交换

这条语句是顺序执行的：

  1. 先执行 a = b：把 a 标签贴到 b 指向的对象上，此时 a 原来的值就丢失了（没有变量引用了）
  2. 再执行 b = a：此时 a 已经指向了原来 b 的对象，所以赋值后 b 还是指向原来的对象，最终两个变量都指向同一个值，无法完成交换

本质区别：a, b = b, a 是先计算所有右边的值，再统一赋值；而分开写是逐行赋值，前一行的赋值会影响后一行的取值。
