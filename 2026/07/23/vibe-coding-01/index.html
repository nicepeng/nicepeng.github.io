01 | Vibe Coding 初体验 —— Blot

Vibe-coding 这个词我经常看到，但是却并没有去深入了解这个词的具体含义。不过，它的确没有精确的定义，大意是：你不需要一行行扣代码，而是把想法、感觉、“氛围”以自然语言的形式描述给 AI，让它帮你把软件“变”出来。

Blot 是一款在线的AI应用生成平台，它不需要安装任何软件，打开浏览器就能使用，而且写完代码立刻就能看到效果，做完之后支持一键发布到网上，完全零门槛。

一、注册Blot

1）打开浏览器，访问 https://bolt.new/

2）点击右上角 “Sign in” 登录按钮

3）登录方式支持 Google 账号 / GitHub 账号 / 邮箱 登录（推荐使用GitHub）

登陆成功后，准备工作就做完了，接下来可以在对话框中输入需求来生成项目。

二、开启第一个 AI 项目

第一个项目，不用太难，做一个简单的小工具即可，最好这个工具能有实际用处，这样有效果反馈才会有动力继续学习。

根据网上某些技术类博主的推荐，第一个 AI 项目可以从以下项目入手：

  • 喝水提醒 / 习惯计数器
    • 点击按钮记录一次喝水/运动，显示今日次数和进度环，每日重置。
  • 简易记账本
    • 添加收入/支出、选择分类（餐饮、交通等）、显示总额和明细列表。
  • 番茄钟 + 任务专注计时
    • 25分钟倒计时，完成一个番茄后记录并显示连续天数。
  • 天气查询工具
    • 输入某个城市，就能实时查询该城市最近一周的天气情况。

这次准备做一个待办事项清单应用，即一个简单的To do List，支持新增、标记、修改、删除任务。

3.3 对话技巧

在与 AI 进行多轮编程对话时，高效沟通的核心是把 AI 当成一个需要清晰上下文和渐进式引导的协作者。以下是我从刚才的交互中提炼出的关键技巧：

先搭骨架，再填血肉

不要一次性扔出全部需求，容易遗漏或逻辑混乱。首轮描述“最小可用产品”（如基础增删改查），后续轮次逐步追加功能（如优先级、渐变背景、子任务）。

使用结构化、模块化的提示词

AI 对分点、分类、加粗标题的响应质量明显更高。用“### 一、视觉增强”这类小节分隔，每个功能写清触发方式、表现效果、数据关系。这样做的好处就是 AI 不会漏实现，也方便你后续删减不需要的模块。

明确约束条件，减少歧义

技术栈、文件结构、存储方式、动画类型等，越具体越稳。直接写“仅用 HTML+CSS+原生 JS，单文件，localStorage 持久化”，避免 AI 自行引入 React 或外部库。

善用“追加/修改”的上下文衔接词

多轮对话时，明确告知这是扩展而非重写。开头写“请在上面的待办任务清单基础上，追加以下功能”，防止 AI 丢弃前文成果。

分阶段验证，及时纠偏

不要等 AI 生成全部功能后再测试。先实现核心功能，运行无误；再追加样式/筛选；再加数据导出……每轮只关注一个层面，出错时缩小排查范围。

用英文提示词处理国际主流模型

如果用的是 ChatGPT、Cursor 等，英文提示词往往比中文更稳定（尤其是技术术语），可以准备中英双语版提示词。

四、部署上线

应用做完了，独乐乐不如众乐乐，把它发布到互联网上，让全球的人都能访问到！

Blot.new 页面上也提供了部署方式，只需要点击页面右上角的 “Publish” 按钮，就能发布成功。

稍等片刻，马上就能在对话框看到发布地址。

部署成功后，在任何一台浏览器都能访问你的应用，在微信上点击链接，也能打开。

恭喜，你的第一个应用就已经发布上线了，赶紧把链接分享给你的好伙伴们体验下吧！

如果想自定义访问域名，需要升级到 Blot Pro 会员，不过没必要！

五、理解你的产品

利用AI Coding 完成了项目，我们得好好理解一下你做的这个项目，这样更有利于日后做出更好的产品。

目前这个“待办任务清单”只涉及到前端技术，用的是前端三件套：HTML、CSS、JavaScript。后期如果想对功能进行扩展，例如引入登录功能、持久化机制等，可以加上后端模块。

六、功能拓展

目前实现的待办任务清单，也只是实现了一个雏形，如果想做得更好，可以试着往以下几个方向继续拓展功能：

6.1 🎯 体验优化：

  • 输入智能解析
    • 在输入框用自然语言快速创建任务，如“明天下午3点 提交报告 !紧急”，自动提取日期、优先级并创建。
  • 智能建议
    • 根据历史任务，输入时自动补全常见任务标题。
  • 时间可视化进度条
    • 对带截止日期的任务，显示剩余时间进度条（如“剩余 3 小时”），颜色随紧迫度变化。
  • 任务置顶
    • 允许将关键任务钉在列表顶部，不受排序规则影响。
  • 批量修改日期
    • 选中多个任务后，统一顺延截止日期（如“全部推迟 1 天”）。
  • 空状态引导
    • 列表为空时显示友好插画和提示，例如“✨ 一切搞定，休息一下吧”，并可一键添加示例任务。

6.2 🧠 智能与自动化：

  • 番茄钟集成
    • 任务卡片内嵌一个简易番茄钟，点击开始专注计时，完成后记录专注次数。
  • 习惯打卡模式
    • 对每日/每周重复任务，用日历热力图展示完成连续性，激励保持习惯。
  • 过期任务自动归档
    • 可设置规则：超过截止日期 3 天未完成的任务自动移入“已过期”视图或回收站。
  • 优先级自动推荐
    • 基于任务标题关键词（如“紧急”“明天”“客户”）和截止日期远近来建议优先级。

6.3 👥 协作与多端

  • 共享清单（只读/可编辑）
    • 生成一个分享链接，他人可通过浏览器查看或协同编辑（可用 WebRTC 或简易后端）。
  • 多设备同步
    • 除了 localStorage，可通过 WebDAV、GitHub Gist 或简易后端实现跨设备同步。
  • 任务评论/讨论
    • 为每个任务添加简短日志或评论，记录进展备注，形成时间线。
  • 指派/负责人
    • 在共享清单中为任务指派负责人头像或名字缩写，清晰分工。

6.4 🎨 个性化与情感化

  • 自定义背景图/壁纸
    • 支持上传背景图片或使用 Unsplash 随机图，让界面更亲切。
  • 音效与振动
    • 完成任务播放清脆音效，移动端删除时可触发触觉反馈（需 Web Haptics API）。
  • 主题市场
    • 预设多套配色方案（森林、海洋、极简等），一键切换整套 CSS 变量。
  • 成就与等级系统
    • 完成一定数量任务后获得徽章或解锁新主题，增加使用乐趣。
  • 每日一句
    • 顶部随机显示名言或自定义座右铭，由用户自行维护句子库。

6.5 📊 数据洞察

  • 完成趋势图
    • 用图表展示近 7/30 天的任务完成数量变化，帮助回顾效率。
  • 时间分配分析
    • 按分类或优先级统计投入任务的时间分布（若集成计时），给出优化建议。
  • 导出漂亮报告
    • 将完成清单导出为格式化的图文摘要或分享卡片，方便截图分享。

以上这些拓展思路，你都可以在聊天框中通过追加提示词的方式，让AI帮你实现。

七、写在最后

第一次用 Bolt.new 把一句话需求变成一个能跑的待办清单，对我来说，最大的冲击不是“它居然能写代码”，而是整个创作流程完全变了。

以前做这样一个 Demo，我需要：

  • 起一个本地项目
  • 写 HTML 骨架
  • 调 CSS 布局
  • 写 JS 交互逻辑
  • 反复调试，直到界面和行为符合预期

但这一次，我做的事情变成了：

  1. 用自然语言把“我想要什么”描述清楚
  2. 看它生成一个可以立刻预览的应用
  3. 不满意的地方，直接说“改这里”“加那个”
  4. 几分钟之内，一个能增删改查、能本地持久化的待办清单就成型了

Vibe-coding 给我的不是“替代编程”，而是“把想法立刻变成原型”的超能力。

它让我在还不太确定交互细节的时候，就能先看到一个“活的东西”。然后我再去调整逻辑、完善边界、思考数据结构。这种“先感性，再理性”的顺序，反而让想法落地的速度成倍提升。

所以，Vibe-coding 给我的不是一个“写代码更快”的捷径，而是一种新的创作心态：大胆想，快速做，持续改。 在 AI 时代，这三件事，比任何时候都更重要。

如果你还没试过，我的建议是：别犹豫，把你脑子里那个一直想做但觉得“太难了”的想法拿出来，丢给 AI，看看会发生什么。
