06 | 列表与元组：从基础用法到底层原理，彻底搞懂 Python 最常用的序列容器

上一篇我们系统讲解了字符串 ——Python 中最常用的字符序列。而在实际开发中，我们需要存储的远不止字符：数字、对象、甚至其他容器，都需要一个通用的有序容器来承载，这就是列表（list）与元组（tuple）。

很多新手刚接触时会疑惑：不都是存一组数据吗？为什么要做两个类型？元组不能修改，存在的意义是什么？为什么同样是拷贝，改嵌套列表会互相影响？

这篇文章我们会从基础用法、核心特性、底层原理、最佳实践四个维度，彻底讲透列表与元组，覆盖从入门到面试的所有核心知识点。

一、核心本质：序列、可变与不可变

1.1 什么是序列（Sequence）

列表和元组都属于 Python 的序列类型，序列是一组有序元素的集合，所有序列都有一套通用的操作规则：

  • 有序性：每个元素有固定的位置索引，从 0 开始编号
  • 共性操作：支持索引访问、切片截取、遍历、长度计算、成员判断等
  • Python 中常见的序列：字符串（str）、列表（list）、元组（tuple）、字节串（bytes）

1.2 列表（list）：动态可变的通用容器

列表是 Python 中最灵活的有序容器，核心特性是可变（Mutable）—— 创建之后，可以随时增加、删除、修改内部元素。

  • 定义方式：用方括号[]包裹，元素之间用逗号分隔
  • 元素规则：支持任意数据类型（数字、字符串、列表、对象等），元素可重复
  • 创建方式：

1.3 元组（tuple）：静态不可变的容器

元组和列表几乎完全对称，核心特性是不可变（Immutable）—— 创建之后，不能增加、删除、修改内部元素。

  • 定义方式：用圆括号()包裹，元素之间用逗号分隔
  • 元素规则：和列表一致，支持任意数据类型，可重复

二、通用序列操作：列表和元组都能用

作为序列类型，列表和元组有 80% 的操作是完全通用的，这些语法和字符串也高度一致，学习成本极低。

2.1 索引访问

通过索引定位单个元素，支持正索引（从 0 开始）和负索引（从 - 1 开始，倒数）。

⚠️ 注意：索引超出范围会直接报 IndexError: list index out of range 错误。

2.2 切片操作

截取序列中的一段子序列，语法和规则和字符串完全一致：序列[start:end:step]

  • 左闭右开规则：包含 start 索引元素，不包含 end 索引元素
  • 参数缺省：start 缺省从头开始，end 缺省到末尾结束，step 缺省为 1
  • 步长为负：从右往左截取，可实现倒序

2.3 拼接与重复

  • +：拼接两个同类型序列，生成一个新序列
  • *：将序列重复 n 次，生成一个新序列

⚠️ 经典坑：列表乘法的引用问题

用*重复列表时，如果元素是可变对象，重复的是内存****引用，不是拷贝。修改一个元素，所有重复的元素都会跟着变：

2.4 成员运算与公共函数

  • in / not in：判断元素是否在序列中，返回布尔值
  • 公共内置函数：
    • len()：返回序列长度
    • max() / min()：返回最大值 / 最小值（元素类型需可比较）
    • sum()：对数字序列求和
    • sorted()：对序列排序，返回新的列表

2.5 遍历方式

普通遍历：只获取元素

带索引遍历：同时获取索引和元素（推荐，非常常用）

三、列表专属操作：可变带来的丰富方法

因为列表是可变对象，所以拥有大量增、删、改的内置方法 —— 这也是列表和元组最大的功能区别。我们按功能分类拆解，每个方法讲清用法、区别和踩坑点。

3.1 增加元素

方法 作用 核心特点
append(item) 末尾追加一个元素 把参数整体当成一个元素追加，会产生嵌套
extend(iterable) 末尾追加可迭代对象的所有元素 把参数拆开，逐个追加到末尾
insert(index, item) 在指定索引位置插入元素 插入位置后所有元素都会后移，性能比尾部追加差

✅ 高频对比：append vs extend

3.2 删除元素

方法 / 语句 作用 核心特点
pop(index=-1) 删除并返回指定索引的元素 默认删除最后一个；有返回值；索引越界报错
remove(item) 删除第一个匹配的元素 按值删除；无返回值；元素不存在会报ValueError
del 语句 删除指定索引 / 切片的元素，或删除整个变量 Python 关键字，不是列表方法
clear() 清空整个列表 列表变成空列表，对象本身还在

3.3 修改元素

单个修改：通过索引直接赋值

批量修改：通过切片赋值，支持不等长替换（删除切片区域，插入新元素）

3.4 查找与统计

index(item, start, end)：查找元素第一次出现的索引，找不到报错

count(item)：统计元素出现的次数

3.5 排序与反转

这里是新手最容易混淆的知识点，重点区分原地操作和返回新对象。

方法 / 函数 作用 核心特点
list.sort(reverse=False) 对列表原地排序 列表方法，直接修改原列表，无返回值（返回 None）
sorted(iterable) 对可迭代对象排序 Python 内置函数，返回全新的排序后列表，原数据不变
list.reverse() 原地反转列表 列表方法，直接修改原列表
list[::-1] 切片反转 返回全新的反转后列表，原列表不变

⚠️ 经典坑：排序的返回值

很多新手会写a = a.sort()，结果得到None—— 因为 sort 是原地修改，没有返回值。

四、元组：不可变的真相与独有价值

很多新手觉得元组就是 “不能改的列表”，除了省点内存没别的用，这是对元组最大的误解。不可变本身就是元组最核心的价值。

4.1 元组支持的操作

因为不可变，元组没有增、删、改的方法，只支持：

  • 所有通用序列操作：索引、切片、拼接、遍历等
  • 两个查询方法：index()、count()，用法和列表完全一致

4.2 不可变的真正含义

⚠️ 经典面试题：元组****的元素真的完全不能改吗？

答案是：元组不可变，指的是元素的「内存引用」不可变；如果元素本身是可变对象，它的内部内容可以修改。

元组里存的不是元素本身，而是元素的内存地址。不可变指的是 “地址不能变”—— 你不能让这个位置指向另一个对象；但如果地址指向的对象本身是可变的（比如列表、字典），修改这个对象的内部内容是允许的。

4.3 元组的核心优势

  1. 性能更优
    1. 元组创建速度比列表快，占用内存更小。
    2. Python 会对长度较小的元组做缓存优化，重复创建时直接复用内存，不需要向系统申请新空间。
  2. 安全性更高
    1. 作为常量数据使用时，不用担心被意外修改；作为函数返回值传递时，比列表更安全。
  3. 可作为字典的键
    1. 字典的键必须是不可变且可哈希的，列表因为可变不能当键；元素全为不可变类型的元组，可以作为字典的键。
  4. 天然支持解包语法
    1. Python 的多变量赋值、函数多返回值，本质都是元组解包，语法简洁优雅。

六、底层原理：动态数组与深浅拷贝

想真正吃透列表和元组，不能只停留在 API 用法，理解底层实现才能应对复杂场景和面试问题。

6.1 底层存储结构

列表：动态数组（Dynamic Array）

列表的底层是一个存储元素引用的****动态数组：

  • 数组里存的不是元素本身，是每个元素的内存地址（指针）
  • 动态扩容：当元素数量超过数组当前容量时，Python 会自动申请一块更大的内存空间，把旧数据复制过去，再释放旧空间
  • 扩容策略：CPython 中扩容比例约为原容量的 1.125 倍，加上少量固定偏移

性能影响：

  • 尾部追加元素（append）：平均时间复杂度 O (1)，效率极高
  • 中间插入 / 删除元素：需要移动插入位置后面的所有元素，时间复杂度 O (n)，数据量大时性能差
  • 索引访问：通过下标直接定位，时间复杂度 O (1)，极快

元组：静态数组

元组的底层也是引用数组，但大小固定，不能扩容。

  • 因为不可变，Python 对元组做了很多缓存优化：长度 0-20 的元组会被缓存，重复创建时直接复用，不需要向系统申请内存
  • 创建速度、访问速度都略优于列表，内存占用更小

6.2 浅拷贝与深拷贝

⚠️ 超级高频考点，也是新手头号大坑

当列表中嵌套了可变对象时，不同的拷贝方式会带来完全不同的结果，很多人踩过坑。

（1）直接赋值：不是拷贝

用=赋值只是给同一个对象贴了新标签，两个变量指向完全相同的内存地址，改一个另一个必然跟着变：

（2）浅拷贝（Shallow Copy）

创建一个新的列表对象，但新列表里的元素，还是原列表元素的引用 —— 只拷贝最外层，内层嵌套的对象不拷贝。

实现方式：切片[:]、list()构造、copy.copy()

（3）深拷贝（Deep Copy）

完全递归拷贝所有层级的对象，新列表和原列表从外到内完全独立，互不影响。

实现方式：copy.deepcopy()

总结与选型

拷贝方式 特点 适用场景
直接赋值 完全共享，同生共死 只是给对象起别名
浅拷贝 只拷贝外层，内层共享 一维列表，元素都是不可变类型
深拷贝 完全拷贝，彻底独立 嵌套了可变对象，需要完全隔离

七、最佳实践与常见避坑

7.1 列表 vs 元组，怎么选？

✅ 选列表：数据需要频繁增删改；需要动态变化的容器

✅ 选元组：数据固定不变，只用于读取遍历；作为字典的键；函数返回多个值；作为常量提升安全性

7.2 循环中删除列表元素的坑

正序遍历列表并删除元素，会导致后续元素前移，漏掉下一个元素，出现漏删：

✅ 正确方案（推荐优先级从高到低）：

列表推导式：生成新列表，最简洁高效

倒序遍历删除：从后往前删，删除元素不影响前面的索引

遍历副本，删原列表：不推荐，浪费内存

7.3 其他避坑点

  • 不要用list、tuple当变量名，会覆盖 Python 内置函数
  • sort()是原地修改，没有返回值，不要写a = a.sort()
  • 列表乘法不要用于嵌套可变对象，用列表推导式替代
  • 元组单个元素别忘了加逗号

详细解析

b = a[:]是浅拷贝，只拷贝最外层列表，内层元素都是引用共享：

  • b[0] = 100：修改外层整数（不可变），相当于给 b [0] 重新赋值，不影响 a，所以 a [0] 还是 1
  • b[1]["age"] = 20：b [1] 和 a [1] 指向同一个字典（可变对象），修改字典内容，a 也会跟着变
  • b[2] += (4,5)：元组是不可变的，+=会生成新元组，重新赋值给 b [2]，不影响 a [2]，所以 a [2] 还是原来的元组

c = deepcopy(a)是深拷贝，所有层级的对象都完全复制，和 a 彻底独立，所以 b 的所有修改都不会影响 c。
